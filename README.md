# A02
## Part 2: Glossary

- **Branch** – A separate version of a project where you can make changes without affecting the main project.
- **Clone** – Making a copy of a repository from GitHub to your computer.
- **Commit** – Saving changes in Git with a message explaining what was changed.
- **Fetch** – Getting updates from GitHub without merging them.
- **GIT** – A version control system used to track changes in files.
- **Github** – A website that stores Git repositories online.
- **Merge** – Combining changes from one branch into another.
- **Merge Conflict** – When two changes cannot be merged automatically and must be fixed manually.
- **Push** – Sending your local changes to GitHub.
- **Pull** – Getting changes from GitHub and updating your local project.
- **Remote** – The online version of your repository.
- **Repository** – A folder that stores your project files and their history.
